# 写作规范 (AI Reference)

本文档旨在为人工智能提供一套标准化的深度专著书籍编写指令。在生成书籍大纲及书籍内容时，请严格遵守以下规范。

## 1. 核心原则 (Core Principles)

生成必须遵循 **“五一原则”**，确保全书逻辑统一，拒绝简单的章节堆砌。

- **五一核心**：
  - **统一结构**：全书架构层次分明。
  - **统一术语**：核心概念定义一致。
  - **统一逻辑**：论述路径连贯。
  - **统一格式**：标题与编号规范。
  - **统一交付**：每章均有明确输出。

- **闭环要求**：
  每个知识单元需完成逻辑闭环：
  1. **为什么**（痛点/背景）
  2. **是什么**（原理/核心概念）
  3. **怎么做**（落地/执行步骤）

- **落地导向**：
  - 避免纯理论。
  - 坚持“通俗化 + 场景化”。
  - 术语必解释，公式必示例。

---

## 2. 结构分层规范 (Structural Hierarchy)

### 宏观架构
推荐采用 **“篇 - 章 - 节”** 三级架构。

### 功能模块组合
根据书籍定位（技术/管理/政策等），从以下四种模块中自由组合：

| 模块类型 | 核心功能 | 回答的问题 |
| :--- | :--- | :--- |
| **1.【理论/制度】型** | **构建认知逻辑**：阐明底层机理 | “为什么” |
| **2.【方法/模型】型** | **提供操作工具**：拆解步骤与公式 | “怎么做” |
| **3.【实战/场景】型** | **解决具体痛点**：连接场景与方法 | “如何落地” |
| **4.【趋势/洞察】型** | **前瞻未来价值**：分析趋势与风险 | “以后会怎么样” |

## 3. 章节内容颗粒度规范 (Content Granularity)

大纲及内容需细化至 **“节”** 级，并根据内容属性应用以下模板。

### 3.1 “篇”级必备
- **篇首语** (300-500字)：
  - 功能：总起、导航。
  - 内容：破题、内容定位、内容导览。

### 3.2 “章”级必备
1. **章引言**（痛点破局法）：
   - 明确核心痛点（设问）。
   - 本章目标（交付成果）。
   - 逻辑地图（带编号的节标题列表）。
2. **本章小结**：
   - 提炼 3-5 条核心观点。
   - 针对不同从业角色的关键建议。

### 3.3 “节”级分类模板 (逻辑零件)

请根据节的内容类型，选择对应的逻辑结构：

- **A. 理论型**
  1. 核心观点（开门见山）
  2. 逻辑演绎（主题分析）
  3. 实践启示

- **B. 方法型**
  1. 定义/目的
  2. 操作路径（步骤/公式/参数）
  3. 实战案例
  4. 关键要点/陷阱提示

- **C. 实战型**
  1. 痛点破局
  2. 逻辑解析（场景特点）
  3. 方法路径
  4. 案例研讨
  5. 落地工具

- **D. 前沿型**
  1. 趋势破题
  2. 深度分析
  3. 案例/实践现状
  4. 战略建议/未来展望

## 4. 逻辑衔接与辅助规范 (Auxiliary Standards)

### 4.1 承上启下机制
为避免“孤岛化”，必须在章节开头、正文及结尾显式引用相关章节。
- *示例*：“基于第X章明确的逻辑...本章将探讨...”

### 4.2 标准库支撑
大纲及内容中应规划以下附录或支撑库：
- **术语表**：统一核心定义 (中英文对照)。
- **案例库**：规范案例编号、脱敏主体、关键结论。
- **政策法规库**：法律渊源、发布单位、适用范围。
- **工具箱**：框架图、决策树、计算模板。
- **简称表**：政策、机构的规范简称。

### 4.3 排版约束
- **层级编号**：六级层级（第一篇 → 第一章 → 第1节 → 一、 → (一) → 1.）
- **标题限制**：不超过 15 字。
- **图表编号**：严格按“章号-序号”编制（如 图 3-1）。
